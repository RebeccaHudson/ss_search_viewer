FAQ

Q1. Is atSNP DB exclusively for human?
  • A1. Our current release only has data from the human genome. We plan to extend the database to mouse in the near future.

Q2. Which db SNP version was used for atSNP db?
  • A2. atSNP DB is based on dbSNP build 144 for Human Build 38 (6Rch38). Summary on the dbSNP build 144 can be found here.

Q3. How does atSNP DB handle SNPs with indels, and others?
  • A3. dbSNP classifies genetic variation in eight types. Click here for the types of variation. The current atSNP DB only considers the major single nucleotide variation class; the class TAKES 88.8% of all genetic variation, specifically 132,946,852 SNPs out of 149,735,377.

Q4. How are multi-allelic SNPs evaluated?
  • A4. atSNP DB considers all possible alleles one at a time and provides atSNP statistics and p-values.

Q5. Which motif libraries were used for atSNP?
  • A5. We used JASPAR vertebrate library and known and discovered motifs for the ENCODE TF ChIP-seq datasets, which are available here for JASPAR, and here for ENCODE, respectively. For users who want to learn more about the libraries, visit the websites or click here

Q6. Can I have summary statistics of atSNP DB?
  • A6. We investigated 132,946,852 SNPs and considered 2,270 PWMs. We have evaluated total 307,602,574,340 SNP-PWM combinations.

Q7. Are downloads available?
  • A7. The current release allows downloading the results up to the first 5,000 rows in CSV format.

Q8. How can I interpret the logo plots?
  • A8. The plots are called composite logo plots and they aim to provide a quick visualization of the region around the SNP and the sequence matches to the PWM. Please click here for the interpretation.

Q9. Why can't I page past 10,000 search results?
  • A9. If more than 10,000 results are returned, only the first 10,000 will be avaiable to page through. To access results past the 10,000th, adjust the sort order and/or priority.

Q10. How do I cite atSNP DB?
  • A10.
    • 1. Zuo, Chandler, Shin, Sunyoung, and Keleş, Sündüz. atSNP: transcription factor binding affinity testing for regulatory SNP detection. Bioinformatics (2015): 31, 3353-3355.
    • 2. Shin, Sunyoung, Hudson, Rebecca, Harrison, Christopher, Craven, Mark, and Keleş, Sündüz. atSNP DB: a web resource for quantifying impact of dbSNP SNPs on transcription factor binding. (2018). In preparation.

Q11. What web browsers are supported by atSNP?
  • A11. Everything except Internet Explorer (newest versions are called 'Edge').

Q12. What is a non-technical interpretation of the three p-values?
  • A12-1. P-value SNP Impact: Is the SNP a candidate for potential gain- or loss-of -function?
  • A12-2. P-value Reference: Is there a candidate binding site at the SNP location with the reference allele?
  • A12-3. P-value SNP: Is there a candidate binding site at the SNP location with the SNP allele?

Q13. How does atSNP DB define gain-of-function?
  • A13. atSNP DB generates candidates for gain of function by identifying (SNP, PWM) pairs with

    P-value SNP Impact <= 0.05
    P-value Reference > 0.05
    P-value SNP <= 0.05

    This essentially identifies (SNP, PWM) pairs which do not have a statistically significant match with the reference allele and the SNP allele leads to a statistically significant match (i.e., creating a binding site). Note that these p-values are only suggestive and users should make them more stringent.

Q14. How does atSNP DB define loss-of-function?
  • A14. atSNP DB generates candidates for loss-of-function by identifying (SNP, PWM) pairs with

    P-value SNP Impact <= 0.05
    P-value Reference <= 0.05
    P-value SNP > 0.05

    This essentially identifies (SNP, PWM) pairs which have a statistically significant match with the reference allele and the SNP allele lead to a statistically insignificant match (i.e., destroying a binding site). Note that these p-values are only suggestive and users should make them more stringent.

Q15. How are motifs (PWMs) classified into low, moderate, high, and very high degeneracy groups?
  • A15. This is based on computing information content of each individual position and then averaging the information content across all positions. Then, each PWM is classified as follows based on its average information content.
  • Information ContentLabel
    [0, 0.5] Low
    [0.5, 2] Moderate
    [2.5, 3] High
    [FOO, BAR]Very High

  • Low information content indicates a very degenerate motif. While this does not have implications in terms of actual functionality, when possible, following up less degenerate motifs is more preferable.

Q16. What is a good use for atSNP DB sort order?
  • A16. Options for sorting the queries are useful for getting top candidates of gain or loss of function. For example, top loss-of-function candidates can be revealed by
    Sort by:

    P-value Ref: Ascending
    P-value Impact: Ascending
    P-value SNP: Descending

    Similarly, For example, top gain-of-function candidates can be revealed by
    Sort by

    P-value SNP: Ascending
    P-value SNP Impact: Ascending
    P-value Reference: Descending

Q17. I would like to evaluate some SNPs that are not in atSNP DB. Any suggestions?
  • A17. We suggest using the atsnp R package. The R package has both the JASPAR and the ENCODE motif libraries that you can evaluate your SNPs against.

Q18. I would like to evaluate my SNPs against PWMs that are not part of JASPAR or ENCODE. Any suggestions?
  • A18. We suggest using the atsnp R package which allows user defined PWMs.

Q19. I am working on data from Diversity Outbred (DO) mouse. Can I use atSNP DB?
  • Q19. The current release of atSNP DB only has human data. We suggest using the atsnp R package for evaluating your DO SNPs.
